高専プロコン

🍱 EAT & GO - イベント管理システム

「EAT & GO」は、お祭りやイベント会場での屋台の行列やごみ処理の問題を解決するために開発された革新的なシステムです。
来場者・出店者・運営側それぞれに利便性をもたらし、持続可能で誰もが快適に楽しむことができる祭りの実現を目指します。
📱 デモを体験する

プロジェクト概要

このプロジェクトは、以下の課題を解決することを目的としています：
• 屋台の長い行列 - スマートフォンでの事前注文により待ち時間を削減
• ごみ処理・分別の負担 - ゲーミング機構を用いた正しい分別の促進
• 決済の非効率性 - キャッシュレス決済による衛生的で効率的な運営
• 環境負荷 - ポイント還元システムで環境配慮を促進
• 運営の複雑性 - 統合ダッシュボードで全体を一元管理

システム動作フロー

① スマートフォンで注文 → ② 注文受付・調理・QRシール発行 → ③ 商品受け取り → ④ ゴミ箱状況確認 → ⑤ QRコードでごみ分別投棄 → ⑥ 正しく分別できればポイント付与 → ⑦ 運営側でゴミ情報確認・回収 → ⑧ ポイント景品交換

ユーザーロール別機能

  • 🎪 来場者向け機能: デジタルメニュー閲覧、スマートカート機能、キャッシュレス決済、リアルタイム通知、ゴミ箱マップ表示、ポイント管理、イベント情報確認
  • 🏪 出店者向け機能: 注文管理画面、QRラベル印刷、売上分析、メニュー管理、在庫管理、営業時間設定、売上レポート
  • 🔧 運営者向け機能: 統合ダッシュボード、ユーザー管理、廃棄物管理、お知らせ管理、マルチ会場対応、分析レポート、システム監視

使用技術 (技術スタック)

  • 言語とフレームワーク:
    • TypeScript
    • Next.js 14 (App Router)
    • React
    • Tailwind CSS
  • バックエンド・データベース:
    • Supabase (PostgreSQL)
    • Prisma ORM
    • Next.js API Routes
  • 外部API・サービス・ツール:
    • PayPay決済API
    • Brevo / SendGrid（メール通知）
    • Vercel（ホスティング）
    • GitHub（バージョン管理）
    • Raspberry Pi（IoT・ゴミ箱監視）

主な実装機能

  • 🛒 ショッピング機能: スマートカート機能、複数支払い方法対応（現金・クレジット・PayPay）、リアルタイム在庫管理、店舗別メニュー表示
  • 🍳 出店者向け機能: QRラベル自動生成（62mm用紙対応）、注文リアルタイム管理、売上分析ダッシュボード、メニュー管理画面
  • 🔧 運営者向け機能: 統合ダッシュボード、ユーザー管理システム、Raspberry Piからのゴミ箱センサー情報をリアルタイム表示、マルチテナント対応、リッチテキストエディター
  • 🎁 ポイント・ゲーミング機構: ポイント自動付与システム、景品交換機能、分別ポイントランキング、環境配慮インセンティブ

データモデル

Prisma ORM による型安全なデータベース設計：
• Users - ユーザー情報（来場者・出店者・運営者）
• Products - 商品情報
• Orders - 注文情報
• OrderItems - 注文明細
• Announcements - イベント情報
• TrashBins - ゴミ箱情報（IoT連携）
• Points - ポイント履歴
• Shops - 出店者情報

セキュリティ機能

  • Supabase Auth - JWT ベースの認証システム
  • Row Level Security (RLS) - データベースレベルのアクセス制御
  • API 検証 - 入力値の厳密な検証
  • ロールベースアクセス制御 - Role-based Access Control
  • 暗号化通信 - HTTPS による全通信の暗号化

工夫した点・苦労した点

  • 複数決済方法への対応: PayPayなどの外部決済APIを統合し、複数の支払い方法をサポートするアーキテクチャを設計。ユーザーの利便性を大幅に向上させました。
  • 認証・セキュリティ: Supabaseを使用したセキュアな認証機能を実装し、ユーザーデータの保護とアクセス制御を確保。
  • リアルタイムデータ同期: 注文状況の更新をリアルタイムで反映させるため、Supabaseのリアルタイム機能を活用。
  • IoT連携: Raspberry Piで監視するゴミ箱センサー情報をWebSocketでリアルタイム同期。
  • マルチテナント対応: 複数会場・複数出店者の同時管理を実現。
  • UI/UXデザイン: Tailwind CSSを活用した効率的で直感的なUI/UXを実装。

開発期間・体制

  • 開発体制: チーム開発(5人)
  • 開発期間: 2025年4月 ~ 2025年10月
  • 開発時間: 約100時間以上

導入効果

  • 📉 待ち時間削減: 従来の30～60分の行列が、スマートフォン事前注文により5～10分に短縮。来場者の満足度向上。
  • ♻️ ゴミ分別改善: ゲーミング機構により分別率が従来の40%から90%以上に向上。環境負荷を大幅削減。
  • 💼 運営効率化: 統合ダッシュボードにより、従来の手作業による集計がリアルタイムに自動化。
  • 💰 売上向上: 事前注文システムにより客単価が15～25%向上。効率的な集計で利益率も改善。
